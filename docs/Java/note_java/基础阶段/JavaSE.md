# JavaSE

## 01-入门

### 1.1-java是什么

- java是一门面向对象的语言
- java的特性：
  - 简单性-没有指针，自动GC
  - 面向对象-java是一种完全面向对象的
  - 高效性-支持多线程
  - 安全性-验证，异常处理
  - 跨平台-一处编译，处处执行，平台就是操作系统，通过JVM实现

### 1.2-程序的运行

1. java的开发环境-JDK
   - 提供了java开发所需的编译器、调试器、java运行环境（jre）、类库...

2. 配置环境变量

   - JAVA_HOME
     - 配置的是JDK的安装目录
     - 告诉系统以及相关的开发工具，JDK的安装位置

   - PATH
     - 配置的是JDK的bin目录
     - 系统执行程序的查找路径
   - 验证
     - java -version
     - 开关机后，环境变量失效；重新配置，配置完成之后重启

3. HelloWorld
   - 编译：javac 文件名
     - 将源代码编译成字节码文件，可以直接在JVM中运行的代码
   - 运行：java  类名
     - 在命令行中打印出运行结果

### 1.3-JDK、JRE、JVM

1. JDK
   - Java的开发工具包
   - 包含开发工具  javac  java、JRE、类库
2. JRE
   - Java运行时环境——Java程序运行的必须环境
   - 包含运行时类库、核心库和扩展库、JVM
3. JVM
   - Java虚拟机，执行Java的字节码程序

## 02-基本语法

### 2.1-关键字

1. 定义：在Java中已经定义好的，具有一定的功能和作用的单词
2. 数量：48个关键字+3个值类型+2个保留字
   - 3个值类型·：true、false、null
   - 2个保留字：goto、const

### 2.2-标识符

1. 定义：在编写Java代码时，定义的一些名字。例：类名、包名、接口名...
2. 命名规则：
   - 只能由字母、数字、下划线、$组成
   - 不能以数字开头
   - 不能使用关键字和保留字
   - 严格区分大小写
   - 注：可以使用汉字作为标识符，因为汉字是使用Unicode编码，但不推荐使用！
3. 命名规范：
   - 标识符，应该由具有指定含义的英语单词组成 
   - 类名、接口名、枚举名：大驼峰命名法
   - 变量名、方法名：小驼峰命名法
   - 常量名、枚举值：全部大写，下划线命名法

### 2.3-变量和常量

1. 变量
   - 在程序运行过程中，值可以改变的量
   - 定义：数据类型 变量名 = 值
   - 注意：
     - 变量必须声明才能使用
     - 变量必须赋值才能使用
     - 变量具有作用域（一个{}之内）
     - 变量不能重复声明
2. 常量
   - 在程序运行过程中，值不可以改变的量
   - 定义：final 数据类型 常量名 = 值
   - 常量与变量的不同，在于常量只能赋值一次
3. 量
   - 内存空间

### 2.4-数据类型

1. 基本数据类型(8种)

   | 类型    | 值     | 占用存储空间 | 表数范围               | 备注                      |
   | ------- | ------ | ------------ | ---------------------- | ------------------------- |
   | byte    | 整型   | 1字节=8bit   | -128~127               |                           |
   | short   | 整型   | 2字节        | -2的15次方~2的15次方-1 |                           |
   | int     | 整型   | 4字节        | -2的31次方~2的31次方-1 |                           |
   | long    | 整型   | 8字节        | -2的63次方~2的63次方-1 | 赋值很长的整数，使用L/l   |
   | float   | 浮点型 | 4字节        |                        | 赋值一个小数时，使用F/f   |
   | double  | 浮点型 | 8字节        |                        |                           |
   | char    | 字符型 | 2字节        |                        | 只能表示一个字符，unicode |
   | boolean | 布尔型 | 1字节        |                        | 只能表示true或false       |

   - 注意
     - 整数常量值，被默认解析为int类型；
     - 小数常量，被默认解析为double类型

2. 引用类型

   - 数组、类、接口、枚举、注解、记录
   - 与基本数据类型不同，变量保存的不是具体的值，而是对象的引用

3. 类型转换

   - 自动类型转换
     - 表示范围小的，可以直接赋值给表示范围大的数据类型
     - byte->short->int->long->float->double
     - char->int->......
     - 注意：
       - byte和short运算时，会自动转换为int再进行运算
       - 变量在赋值时，不能超出变量范围
       - 为char类型赋值一个整数时，会当作unicode的编码进行处理，范围0-65535
       - 常量值运算，会在编译阶段运行    char c3  = 10 + 20;  即 char c3 = 30;
   - 强制类型转换
     - 范围大的值，通过强制类型转换符转换成范围小的类型
     - 注意：
       - 强制类型转换时，可能会导致数据的溢出

### 2.5-输入输出

1. 输出
   - System.out.print()
   - System.out.println()  换行
   - out 可以换成 err，此时会变成错误输出
2. 输入
   - Scanner scanner = new Scanner(System.in)
   - scanner.next()
   - scanner.close()

### 2.6-注释

1. 单行注释
   - 格式：//
2. 多行注释
   - 格式：/* */
3. 文档注释
   - 格式：/** */
   - 可以使用  javadoc -d mydoc -author -version HelloWorld.java  生成html文档

## 03-运算符与表达式

### 3.1-算术运算符

1. +
   - 数值型的加法运算（整型+浮点型+char）
   - 表示数学上的正号
   - 对字符串进行拼接
2. -
   - 数值型的减法运算
   - 表示数学上的负号
3. *
   - 乘法运算
4. /
   - 除法运算
   - 整数相除=整数
5. %
   - 取余、求模：取除法的余数
   - 对负数进行求余，结果的符号由第一个操作数来决定
   - 对小数进行求余，整数商，余数
6. ++
   - 自增
7. --
   - 自减
8. 前置后置
   - 前置：先运算后赋值
   - 后置：先赋值后运算

### 3.2-比较运算符

1.  < > <= >=
   - 只能对数值型进行比较，比较结果为布尔型
2. == !=
   - 针对8种基本数据类型，比较值是否相等
   - 针对引用数据类型，比较两个引用是否指向同一块内存地址

### 3.3-逻辑运算符

1. 类型
   - 逻辑与：&& 、&
     - 有一个为假便为假
   - 逻辑或：|| 、|
     - 有一个为真即为真
   - 逻辑非
     - ！
   - 短路运算：针对于&& 、||
2. 注：
   - 只针对布尔类型进行运算，运算结果也是布尔类型
   - &&和&区别：&&前面为假，不计算后面
   - ||和|区别：||前面为真，不计算后面

### 3.4-位运算符

1. 按位与
   - 当两个操作位的对应位都是1，结果的对应位也是1，否则为0；有0为0
   - 求负数的二进制：负数的绝对值-1再取反
   - 十进制转换为二进制，模2余法，除以2取余，余数逆排序
2. 按位或
   - 当两个操作位的对应位都是0，结果的对应位也是0，否则为1；有1为1
   - 出现负数时，加负号，取反
3. 按位异或
   - 当两个操作位的对应位不同时，结果为1，相同为0
4. 按位取反
   - 符号：~，0转为1、1转为0
   - 单目运算符：自增、自减、逻辑非、按位取反
5. 左移位  A<<2
   - 所有的二进制位全部向左移动指定的位数，右侧补0
   - 向左移动n位，相当于原数值*2的n次方
6. 右移位 A>>2
   - 所有的二进制位全部向右移动指定的位数，左侧补符号位
7. 无符号右移位  A>>>2
   - 所有的二进制位全部向右移动指定的位数，左侧补0

### 3.5-赋值运算符

1. 符号：=

2. 赋值运算符可以与常见的二目运算符结合成为复合赋值运算符

   - +=  -=  %=  *=  /=

   - 复合赋值运算符在运算时，会自行的对数据类型进行转换

     ```java
     short i = 0;
      i += 1;  //可以
      i = i + 1; //不可以
     ```

### 3.6-三目运算符

1. 符号：A ? B : C
   - A是一个布尔型
   - A为true，执行B；A为false，执行C

2. 三目运算符和赋值运算符
   - 结合性是从右向左的

## 04-流程控制

### 4.1-程序执行的三种结构

1. 顺序结构
   - 代码从前往后，一行一行的顺序执行，最基本的执行方式

2. 选择分支
   - 根据一定的条件进行判定；条件成立时，执行一部分代码；条件不成立时，执行另一部分代码
   - Java中有两种分支结构，if...else...  /  switch case
3. 循环分支
   - 根据一定的条件进行判定，条件成立，重复执行同一段代码；条件不成立，结束循环
   - Java中有三种分支结构，while  /  do...while  /  for

### 4.2-选择分支结构

1. if...else...

   - 语法：

     ```java
     if(条件表达式){
     	代码块
     }else if(条件表达式){
     	代码块
     }else{
     	代码块
     }
     ```

   - 执行过程

     - 计算条件表达式的值
     - 条件成立时，执行对应代码块
     - 条件均不成立时，执行else中的代码块

2. switch...case...

   - 语法：

     ```java
     switch(表达式){
         case 常量值1:
             代码块
             break;
         case 常量值2:
         	代码块
         	break;
         default:
         	代码块：
         	break;
     }
     ```

   - 表达式：byte 、short、int、char、String、enum

   - 执行过程：

     - 计算表达式的值
     - 与case的常量值进行比对
     - 如果相等，执行case后的代码块，逐行执行，遇到break退出
     - 所有都不相等，则执行default的代码块

   - 注意：

     - case后的常量值不能重复
     - 进入case后，遇到break才会结束
     - 若没有break，可能会进入其他case或default

### 4.3-循环分支结构

1. while循环

   - 语法结构

     ```java
     while(表达式){
     	循环体
     }
     ```

   - 执行过程

     - 执行条件表达式，是一个布尔类型
     - 条件成立，进入循环体
     - 条件不成立，退出循环

2. do...while

   - 语法结构

     ```java
     do{
     	循环体
     }while(条件表达式);
     ```

   - 执行过程

     - 执行循环体
     - 循环体执行完成之后，判断条件表达式的值
     - 条件成立，重复执行循环体；不成立，退出循环

   - 注意：

     - 必须以分号结束

   - while和do...while的区别

     - 语法结构：while是条件在前，循环体在后；do...while是循环体在前，条件在后
     - 执行顺序：while是先判断再执行循环体；do...while会先执行一遍循环体
     - 执行结果：while的循环体可能一次也不会执行；do...while至少执行一次

3. for

   - 语法结构

     ```java
     for(初始化表达式;条件表达式;迭代表达式){
     	循环体
     }
     
     /*
     初始化表达式：
         在for循环执行之前执行，对循环进行初始化
         通常声明循环的迭代变量，并初始化一个值
     迭代表达式：
         每次循环体执行完成之后执行，对循环的状态进行控制
         通常在迭代表达式中对迭代变量进行值的改变
     */
     ```

   - 执行过程

     - 执行初始化表达式
     - 判断条件表达式的值
       - 条件成立，进入循环体，循环体执行结束后，执行迭代表达式
       - 条件不成立，执行迭代表达式
     - 迭代表达式执行完成后，判断条件表达式

### 4.4-特殊的循环

1. 嵌套循环

   - 在一个循环中，包含了其他的循环

2. 无限循环

   - 循环条件永远成立

   - ```
     for(byte i = 0; i < 10; i--){}
     // 不是死循环
     // byte是有界的，-128-1=127
     ```

### 4.5-循环的控制

1. break
   - 用在switch中，结束switch
   - 用在循环中，结束循环
2. continue
   - 用在循环中，结束本次循环，直接进入下一次循环
3. return
   - 用来结束方法
   - 为方法提供返回值

## 05-方法

### 5.1-什么是方法

1. 方法就是一组代码的集合
2. 为了实现代码的复用，重复使用。

### 5.2-定义方法

1. 语法：
   - 必须定义在类里面
   - 修饰符  返回值类型   方法名(参数列表){  方法体   }
2. 修饰符
   - 修饰符使用来限定方法的使用范围和使用方式的
   - 限定使用范围：
     - public — 公共的，其他包的非子类
     - protected — 其他包的字类
     - 缺省 — 当前包中
     - private — 私有的，当前类中
   - 限定使用方式：
     - static — 静态的
     - abstract — 抽象的
     - final — 最终的（不能被重写）
     - synchronized — 同步的
     - native — 本地的（用来协调底层C、C++的方法）
3. 返回值类型
   - void：表示方法没有返回值
   - 数据类型：表示方法在结束的每一个出口都要提供相应类型的返回值（注意自动类型提示）
   - return：返回值是由return提供的，如果没有返回值，可以使用return结束方法
4. 方法名
   - 方法名是一个标识符
   - 规范：小驼峰命名法
5. 参数列表
   - 向方法传递数据的载体
   - 参数列表
     - 没有参数：参数列表为空
     - 有参数：指定参数的数据类型和参数的名字，多个参数使用逗号分割

### 5.3-调用方法

1. 调用
   - 使用方法名+()；若有参数，则传递对应参数
   - 语法：方法名( 参数 )
   - 传递的参数：数量和数据类型必须与形参一致

2. 实参和形参
   - 形参：形式参数，方法定义时，参数列表中的参数
   - 实参：实际参数，调用方法时，传递的具体的值
   - 传递机制：值传递，基本数据类型传递数据值，引用数据类型传递地址值

### 5.4-递归

1. 在方法的内部，调用了方法自己
2. 递归可以转换为循环
3. 递归层次过多，会出现 java.lang.StackOverflowError
   - 当方法被调用时，方法将会入栈，栈是一块内存，是有限的
   - 因此递归过多会溢出

### 5.5-重载

1. 概念
   - 在一个类中
   - 方法名相同
   - 参数列表不同
     - 参数的数量不同
     - 参数的数据类型不同
     - 参数的顺序不同
2. 调用
   - 调用重载方法时，根据实参的数据类型，寻找最为匹配的方法进行调用
3. 不构成重载的条件
   - 参数名不同
   - 返回值类型不同
